### 1. openUBMC 智能日志分析 Agent 平台 - 项目牵头 · 核心开发

#### Agent 编排 · 双形态按任务长度路由
- 短链 QA 走 **ReAct**（基于 LangGraph `create_react_agent` + 会话 Checkpointer，支持断点续跑）
- 长链根因走基于 LangGraph 实现的 **Plan-and-Execute** 工作流（Planner 规划 → Executor 分步执行 → Replanner 条件边自反思重规划），解决涉密代码只能提供头文件时 ReAct 易陷入死循环的问题
- 在自建 30 条根因回归集上做 A/B：Plan-Execute 多花约 50% token，换来 Task Completion 从 70% → 82%（**+12 pp**）
- 据此形成"短链走 ReAct、长链走 Plan-Execute"的成本-准确率路由策略

#### RAG 检索 · 混合三路 + RRF 融合
- 设计 HybridRetriever 三路并行（TreeSitter AST 精确 + Qwen Embedding v4 在线向量 + ChromaDB FTS 关键词）+ RRF (k=60) 重排；对比加权求和的优势在于向量距离与关键词得分的 scale 不归一，rank-based 融合更合理
- 内置中英技术术语双向扩展解决"热插拔 ↔ hotplug"语义漂移
- 30+ 场景代码检索回归集：File@5 = 84%、Recall@5 ≈ 78%
- RRF 相比简单拼接 **+6 pp**、查询扩展对中英混杂 query 再 **+9 pp**
- 检索 p95 < 420 ms；索引重建从本地 BGE-M3 的 4h+ 优化至在线服务的 < 20 min

#### Tool Use + Reflection 闭环
- LangChain `@tool` + Pydantic Schema 封装 7 个领域工具（日志解析 / 组件匹配 / 代码定位等）
- JSON 结构化 I/O 直连 ReAct Observation，工具调用成功率 97.3%、Schema 级错误 0
- 配套 LogAnalysisReflector / Replanner 按"代码覆盖率 / 根因具体性 / 调用链完整性"三维打分触发重规划
- overall_score 0.64 → 0.72（**+8 pp**）、5W1H 完整性 **+12 pp**

#### 评估与交付 · Evals-driven + 流式全栈
- 自建 evaluation/ 四层框架：方案质量 0.30（最影响开发者体验）→ 定位 0.25 + 5W1H 0.25（覆盖排查流程）→ 根因 0.20（微服务架构下日志链路不一定完整），加权 overall_score 阈值 0.6 判 pass
- 回归集覆盖单函数 / 单组件 / 跨组件多维度场景，pass 率 76%、类别命中 85%、文件级定位 81%
- 评估采用出题与考试模型隔离设计（GLM4.5 出题 vs Qwen-3 考试），避免自我评估偏差
- FastAPI + SSE 推送 7 类流式事件，首 token ≈ 550 ms、深度分析 p95 ≈ 22 s
- React + Monaco 前端 + 多 LLM provider，本地 → 远端 embedding Fallback > 99%
